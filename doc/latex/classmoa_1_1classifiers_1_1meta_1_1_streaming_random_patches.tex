\doxysubsection{Detailed Description}
Streaming Random Patches

Streaming Random Patches (S\+RP). This ensemble method uses a hoeffding tree by default, but it can be used with any other base model (differently from random forest variations). This algorithm can be used to simulate bagging or random subspaces, see parameter -\/t. The default algorithm uses both bagging and random subspaces, namely Random Patches.

See details in\+:~\newline
 Heitor Murilo Gomes, Jesse Read, Albert Bifet. Streaming Random Patches for Evolving Data Stream Classification. I\+E\+EE International Conference on Data Mining (I\+C\+DM), 2019.

Parameters\+:


\begin{DoxyItemize}
\item -\/l \+: ClassiÔ¨Åer to train. Default to a Hoeffding Tree, but it is not restricted to decision trees. 
\item -\/s \+: The number of learners in the ensemble. 
\item -\/o \+: How the number of features is interpreted (4 options)\+: \char`\"{}\+Specified m (integer value)\char`\"{}, \char`\"{}sqrt(\+M)+1\char`\"{}, \char`\"{}\+M-\/(sqrt(\+M)+1)\char`\"{}. 
\item -\/m \+: Number of features allowed considered for each split. Negative values corresponds to M -\/ m. 
\item -\/t \+: The training method to use\+: Random Patches, Random Subspaces or Bagging. 
\item -\/a \+: The lambda value for the poisson distribution (used to emulate bagging). 
\item -\/x \+: Change detector for drifts and its parameters. 
\item -\/p \+: Change detector for warnings. 
\item -\/w \+: Should use weighted voting? 
\item -\/u \+: Should use drift detection? If disabled, then the bkg learner is also disabled. 
\item -\/q \+: Should use bkg learner? If disabled, then trees are reset immediately. 
\end{DoxyItemize}
